Unidad 1: Conceptos de Ciberseguridad

Conceptos Básicos

Concepto Descripción / Ejemplo
Confidencialidad

Mantener la información accesible solo para quienes estén autorizados.

Ejemplo: cifrado de mensajes y control de acceso a carpetas.
Integridad

Garantizar que los datos no se alteren sin autorización y se pueda detectar cualquier cambio.

Ejemplo: firmas digitales y checksums (hashes).
Disponibilidad

Que los sistemas y datos estén disponibles cuando los usuarios los necesiten.

Ejemplo: redundancia, backups y mitigación de DDoS.
Autenticación

Verificar que una identidad (usuario, dispositivo) es quien dice ser.

Ejemplo: contraseña + 2FA (doble factor).
Autorización

Determinar qué acciones puede realizar una identidad autenticada.

Ejemplo: permisos de archivo y roles en un sistema.

Tipos de Malware (detallado)

Tipo Descripción
Virus

Programa que se adjunta a archivos ejecutables y se propaga cuando se ejecutan.

Impacto: corrupción o pérdida de información; típico en archivos compartidos.
Gusano (Worm)

Se replica de forma autónoma a través de redes explotando vulnerabilidades.

Impacto: congestión de red, caídas de servicios.
Caballo de Troya (Trojan)

Se presenta como software legítimo pero incluye funciones maliciosas ocultas.

Impacto: acceso remoto, robo de credenciales y puertas traseras.
Ransomware

Cifra archivos o bloquea sistemas pidiendo rescate a cambio de la clave de recuperación.

Impacto: interrupción total de operaciones y pérdidas económicas.
Spyware

Monitorea y envía información del usuario sin su consentimiento.

Impacto: violación de la privacidad, robo de datos sensibles.
Adware

Software que inyecta anuncios; a veces acompañando otras amenazas.

Impacto: molestia UX y disminución de rendimiento.
Rootkit

Tecnologías diseñadas para ocultar procesos y mantener acceso persistente.

Impacto: detección difícil y control profundo del sistema.
Keylogger

Registra las teclas que presiona el usuario para obtener contraseñas y datos.

Impacto: robo de credenciales bancarias y cuentas.
Botnet

Red de equipos infectados controlados por un atacante para realizar tareas coordinadas.

Impacto: DDoS, envío masivo de spam, fraude.
Cryptojacker

Software que utiliza recursos del equipo para minar criptomonedas sin permiso.

Impacto: ralentización, sobrecalentamiento y mayor consumo energético.
Wiper

Malware diseñado para destruir o sobrescribir datos de forma irreversible.

Impacto: pérdida masiva e irreparable de información.

Otros conceptos clave

Concepto Descripción / Prevención
Phishing

Mensajes que suplantan identidades para robar credenciales o datos.

Prevención: comprobar remitente, URLs, activar 2FA y educar usuarios.
DDoS

Ataque que satura recursos de un servicio con tráfico masivo.

Prevención: balanceo, filtros y servicios anti-DDoS.
Firewall

Filtra tráfico de red según reglas; actúa como primera línea de defensa.

Buena práctica: reglas mínimas necesarias y monitorización.
Antivirus / EDR

Herramientas que detectan y mitiguan malware (firma / heurística / comportamiento).

Actualizar firmas y combinar con detección de comportamiento.
Backup

Copia periódica e independiente de datos críticos para recuperación.

Regla 3-2-1: 3 copias, 2 medios distintos, 1 offsite.
Ingeniería Social

Técnicas que manipulan a personas para obtener acceso o datos.

Prevención: formación y políticas de verificación.
VPN

Túnel cifrado que protege tráfico en redes públicas.

Usar proveedores confiables y protocolos seguros.
